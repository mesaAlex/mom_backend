API Documentation

Welcome to the Meals on Mesa REST API. Use the endpoints below to access recipe data for the React client application.

GET /api/recipes

Returns an array of all recipes, each containing an id, title, description, image path, tags, timing, servings, calories, ingredients, and instructions.

Example Response

[ { "_id": "...", "title": "Salmon Salad", "description": "A fresh, high-protein salad …", "image": "/images/salmon_salad.png", "tags": ["High-protein", "Low-carb", "Gut-friendly"], "prepMinutes": 10, "cookMinutes": 12, "servings": 2, "calories": 420, "ingredients": ["2 Salmon Fillets", "…"], "instructions": ["Preheat the oven …", "…"] }, … ]
GET /api/recipes/:id

Returns a single recipe object by its MongoDB _id. Returns a 404 JSON error if the recipe is not found.

Example Response

{ "_id": "...", "title": "Salmon Salad", … }
POST /api/recipes

Creates a new recipe. Accepts multipart/form-data with an optional img file field. All text fields are validated with Joi before saving. Returns the created recipe with status 201.

PUT /api/recipes/:id

Updates an existing recipe by _id. Accepts multipart/form-data. If no new image is uploaded, the existing image is preserved. Returns the updated recipe with status 200.

DELETE /api/recipes/:id

Permanently deletes a recipe from MongoDB by _id. Returns { "message": "Recipe deleted successfully" } on success.

GET /images/:filename

Static file serving. All recipe images are available at /images/<filename>.
